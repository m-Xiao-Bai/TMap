# 城市轨道交通查询与票务管理系统开发文档

## 一、项目概述

### 1.1 项目名称
城市轨道交通查询与票务管理系统（可扩展至高铁）

### 1.2 项目背景
随着城市轨道交通的发展，用户对出行效率和购票便捷性的需求日益增加。本系统旨在提供便捷的地铁线路查询、购票、退票功能，并提供多层级管理员管理后台，为系统的稳定运行和数据安全提供保障。

### 1.3 项目目标
- 实现地铁线路查询与路径规划  
- 提供在线购票和退票服务  
- 提供多角色权限管理系统  
- 前端界面美观、交互友好  
- 后端采用高可维护性架构  
- 支持未来扩展（高铁系统、多城市线路）  

---

## 二、技术架构

### 2.1 前端技术
- Vue 3  
- Vue Router  
- Pinia（状态管理）  
- Axios（请求库）  
- Element Plus / 自定义UI组件  
- 模块化组件开发  

### 2.2 后端技术
- JDK 17  
- Spring Boot  
- MyBatis / JPA  
- Redis（Token存储）  
- MySQL（关系型数据库）  

### 2.3 架构模式
- 前后端分离架构  
- RESTful API  
- Token认证机制（Redis存储）  
- RBAC角色权限控制  

---

## 三、系统角色设计

| 角色 | 权限说明 |
|------|---------|
| 普通用户 | 查询线路、购票、退票 |
| 管理员 | 用户管理（启用/禁用） |
| 超级管理员 | 管理员权限 + 地铁站与线路管理 |
| 最高级管理员 | 数据库全局操作（增删改查，批量操作） |

> 每种用户有独立登录页面，用户注册功能仅对普通用户开放。

---

## 四、功能模块设计

### 4.1 普通用户模块
- 注册 / 登录 / 登出  
- 地铁线路查询（可选起点、终点、途径站）  
- 最短路径推荐（Dijkstra算法）  
- 在线购票、退票  
- 查看历史订单  
- 收藏常用线路（扩展功能）  
- 页面包括：登录页、注册页、线路查询页、购票页、我的订单页  

### 4.2 管理员模块
- 用户列表查询  
- 用户状态控制（启用/禁用）  
- 页面包括：管理员登录页、用户管理页  

### 4.3 超级管理员模块
- 地铁站管理（新增/修改/删除/状态码管理）  
- 地铁线路管理（新增/修改/删除/状态码管理）  
- 页面包括：超级管理员登录页、地铁站管理页、线路管理页  

### 4.4 最高级管理员模块
- 数据库全局操作（增删改查）  
- 批量导入/导出数据  
- 系统配置管理（票价规则、Token过期时间等）  
- 日志查看和Redis缓存管理  
- 页面包括：高级管理员登录页、系统控制台  

---

## 五、登录与权限认证设计

### 5.1 Token机制
- 登录成功生成Token  
- Token存储于Redis  
- 前端请求Header携带Token进行验证  

### 5.2 权限控制
- 基于RBAC模型，不同角色访问不同接口  
- Token过期自动失效

## 六、数据库设计（已完成）

### 核心表
- `user`：用户表  
- `station`：站点表  
- `line`：线路表  
- `ticket`：票务表  
- `order`：订单表  

### 数据库设计原则
- 使用状态码管理数据状态（如启用、禁用、维护）  
- 禁止硬编码  
- 可扩展为高铁线路及多城市线路
